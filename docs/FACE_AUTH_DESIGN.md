# 人脸验证流程详细设计文档 v1.0

**作者**：PM
**日期**：2026-05-19
**版本**：v1.0（初稿）
**适用页面**：`app/lib/pages/face_auth_page.dart`

---

## 0. 设计原则

1. **每一步都有"成功反馈停顿"**：眨眼成功 → 显示"识别成功"+ 停顿 1 秒 → 再进入下一步。不无缝衔接。
2. **超时一律退回登录页**：不降级、不切短信。让用户回登录页自己选。
3. **不涉及智能代理**：本文档不出现任何"小浙"语音或代理介入。
4. **逐帧详细**：每个状态独立编号，明确布局、文案、时长、转场。

---

## 1. 状态总览

```
S0 准备页
   ↓ 点击"开始认证"
S1 应用内授权弹窗
   ↓ 点击"同意并继续"
S2 系统权限弹窗（mock）
   ↓ 点击"允许"
S3 摄像头初始化（loading）
   ↓ 摄像头打开成功
S4 面部对位（实时纠正）
   ↓ 对准稳定 1 秒
S5 眨眼检测中
   ↓ 检测到眨眼
S6 眨眼成功反馈（停顿 1 秒）
   ↓
S7 摇头检测中
   ↓ 检测到摇头
S8 摇头成功反馈（停顿 1 秒）
   ↓
S9 全部成功反馈（停顿 1.5 秒）
   ↓
S10 跳转长辈版首页

异常出口：
E1 摄像头被拒绝 → 摄像头权限拒绝页
E2 检测超时（S4/S5/S7 任一超过 20 秒）→ 退回登录页 + Toast 提示
E3 用户主动点 × → 退回登录页
E4 模型加载失败 → 退回登录页 + Toast 提示
```

### S4 · 面部对位（实时纠正）

**触发**：S3 摄像头就绪。

**目标**：引导用户把脸放到圆圈中央，距离合适。

**布局**：
```
┌─────────────────────────────────┐
│ [×]                             │
│                                 │
│        ╭─ ─ ─ ─ ─ ─ ─╮          │
│       │              │          │
│      │  [摄像头预览]  │         │ ← 圆形预览框 280dp
│      │   实时画面    │          │   外圈橙色虚线
│       │              │          │
│        ╰─ ─ ─ ─ ─ ─ ─╯          │
│                                 │
│      请把脸放到圆圈里           │ ← 24sp Bold 主提示
│                                 │
│      对准了我会自动开始         │ ← 18sp 副提示（根据状态切换）
│                                 │
│                                 │
└─────────────────────────────────┘
```

**主提示**：固定为"请把脸放到圆圈里"。

**副提示**（根据 MediaPipe 实时检测切换，**每条都是 18sp**）：

| 检测状态 | 副提示文案 | 颜色 |
|---|---|---|
| 找不到人脸 | "找不到您的脸，请正对屏幕" | 灰色 `#666666` |
| 人脸偏左（脸中心 x < 框中心 - 0.15W） | "请把脸往右移一点" | 灰色 |
| 人脸偏右 | "请把脸往左移一点" | 灰色 |
| 人脸偏上 | "请把脸往下移一点" | 灰色 |
| 人脸偏下 | "请把脸往上移一点" | 灰色 |
| 人脸太近（脸宽 > 框宽 × 0.8） | "请离远一点" | 灰色 |
| 人脸太远（脸宽 < 框宽 × 0.4） | "请离近一点" | 灰色 |
| 光线不足（亮度均值 < 阈值） | "光线太暗，请到亮一点的地方" | 橙色 |
| 已对准 | "**很好，保持不动**" | 绿色 `#4CAF50` |

**对准判定**：人脸中心在框中心 ±15% 范围内 + 脸宽占框宽 40-80% + 亮度达标，**连续 1.0 秒满足**视为对准成功。

**对准过程视觉反馈**：
- 未对准：圆形虚线外圈，颜色橙色 `#FF6D00`
- 对准瞬间：外圈从虚线渐变为实线，转绿色 `#4CAF50`，200ms 渐变

**超时**：S4 进入后 **20 秒未完成对准** → 进入 E2（超时退回登录页）

**交互**：× 按钮 → E3

**转场到 S5**：对准稳定 1 秒后，自动进入 S5，转场用 200ms 淡入新主提示。外圈进度环显现（从 0% 起，外圈套一层橙色进度环）。

### S5 · 眨眼检测中

**触发**：S4 对准成功 + 1 秒稳定。

**目标**：等待用户眨眼一次。

**布局**：
```
┌─────────────────────────────────┐
│ [×]                             │
│                                 │
│        ╭─[█████░░░░░░]─╮         │ ← 进度环外圈 0→50%
│       │              │          │
│      │  [摄像头预览]  │         │ ← 圆形预览框 280dp
│      │   实时画面    │          │   外圈实线绿色
│       │              │          │
│        ╰──────────────╯         │
│                                 │
│       请眨一下眼睛              │ ← 24sp Bold 绿色主提示
│                                 │
│       自然地眨一下就行          │ ← 18sp 灰色副提示
│                                 │
└─────────────────────────────────┘
```

**主提示**：根据时长升级
| 时长 | 主提示 | 颜色 |
|---|---|---|
| 0-5 秒 | "请眨一下眼睛" | 绿色 `#4CAF50` |
| 5-10 秒 | "请眨一下眼睛，自然就行" | 橙色 `#FF6D00` |
| 10-20 秒 | "请用力眨一下眼，像睡着一样" | 橙色 |
| 20 秒 | → E2（超时退回登录页） | — |

**副提示**：固定"自然地眨一下就行"。

**检测逻辑**：MediaPipe FaceLandmarker 计算左右眼 EAR（Eye Aspect Ratio）：
- EAR < 0.20 持续 ≥ 1 帧（约 33ms）且随后恢复到 EAR > 0.25 → 视为完成一次眨眼

**进度环**：固定停在 0%，**不要随时长涨**（避免用户以为时间到了就失败）。直接绑定"完成一次眨眼"事件，瞬间从 0% 跳到 50%（200ms 缓动）。

**超时**：S5 进入后 **20 秒未检测到眨眼** → 进入 E2。

**交互**：× 按钮 → E3

**转场到 S6**：检测到眨眼瞬间，进度环跳到 50%，主提示淡出（150ms），切换 S6。

### S6 · 眨眼成功反馈（停顿）

**触发**：S5 检测到眨眼。

**目标**：明确告诉用户"这一步过了"，让用户看到结果再继续。**这是关键停顿步骤**。

**布局**：
```
┌─────────────────────────────────┐
│ [×]                             │
│                                 │
│        ╭─[██████░░░░░░]─╮        │ ← 进度环 50%（绿色）
│       │              │          │
│      │  [摄像头预览]  │         │
│      │   ▓▓ ✓ ▓▓     │          │ ← 中央叠加一个大绿色对勾
│       │              │          │   80dp，淡入 200ms
│        ╰──────────────╯         │
│                                 │
│        识别成功                 │ ← 28sp Bold 绿色 `#4CAF50`
│                                 │
│      准备进行下一步…            │ ← 18sp 灰色副提示
│                                 │
└─────────────────────────────────┘
```

**文案**：
- 主提示："识别成功"
- 副提示："准备进行下一步…"

**视觉反馈**：
- 摄像头预览中央叠加绿色 ✓ 对勾，背景半透明白色圆形（淡入 200ms）
- 进度环 50% 处停留
- 可选：成功音效（短促"叮"声 200ms）

**持续时间**：**1.0 秒**（用户能看清反馈、感受到"过了一关"的节奏感）。

**转场到 S7**：1 秒后对勾淡出（200ms），主副提示切换 S7 文案。

### S7 · 摇头检测中

**触发**：S6 停顿结束。

**目标**：等待用户左右转头。

**布局**：
```
┌─────────────────────────────────┐
│ [×]                             │
│                                 │
│        ╭─[██████████░░]─╮       │ ← 进度环 50→100%
│       │              │          │
│      │  [摄像头预览]  │         │
│      │   实时画面    │          │
│       │              │          │
│        ╰──────────────╯         │
│                                 │
│   请慢慢往左转头，再往右转      │ ← 24sp Bold 绿色主提示
│                                 │
│       不用幅度太大              │ ← 18sp 灰色副提示
│                                 │
└─────────────────────────────────┘
```

**主提示**：根据时长升级
| 时长 | 主提示 | 颜色 |
|---|---|---|
| 0-5 秒 | "请慢慢往左转头，再往右转" | 绿色 |
| 5-10 秒 | "请慢慢往左转头一下，再往右转一下" | 橙色 |
| 10-20 秒 | "幅度可以小一点，左右转一下头就行" | 橙色 |
| 20 秒 | → E2（超时退回登录页） | — |

**副提示**：固定"不用幅度太大"。

**检测逻辑**：MediaPipe FaceLandmarker yaw 角度
- yaw 达到 ≤ -15°（左转）→ 标记"左完成"
- 然后 yaw 达到 ≥ +15°（右转）→ 标记"右完成"
- 两个标记都完成 → 摇头检测成功
- 顺序可以反（先右再左也行）
- 阈值 ±15° 为保守值（老年人活动范围有限），可由 architect 后续调优

**进度环**：固定停在 50%，**不要随时长涨**。绑定"摇头完成"事件，瞬间从 50% 跳到 100%（200ms 缓动）。

**超时**：20 秒未完成 → E2。

**交互**：× 按钮 → E3

**转场到 S8**：摇头完成瞬间，进度环 100%，主提示淡出，切换 S8。

- 可选：成功音效（较长"叮咚"约 400ms）

**持续时间**：**1.5 秒**。

**转场到 S10**：1.5 秒后页面整体淡出（300ms）+ 路由跳转。

---

### S10 · 跳转长辈版首页

**触发**：S9 持续时间结束。

**操作**（按顺序）：
1. 调用 `ref.read(loginProvider.notifier).login('用户')` 写入登录态
2. 释放摄像头资源（`MediaStream.getTracks().forEach(track => track.stop())`）
3. 释放 MediaPipe FaceLandmarker 实例
4. `context.go(AppRoutes.elderHome)` 跳转长辈版首页

**用户看到的**：S9 页面淡出 300ms 后，长辈版首页淡入 300ms。

## 3. 异常分支详细设计

### E1 · 摄像头被拒绝页

**触发**：S2 用户点"禁止" 或 任何时候 getUserMedia 抛 NotAllowedError。

**布局**：
```
┌─────────────────────────────────┐
│ [×]                             │
│                                 │
│        ┌──────────────┐         │
│        │   ⚠️          │         │ ← 橙色警告图标 64dp
│        └──────────────┘         │
│                                 │
│       无法打开摄像头            │ ← 24sp Bold
│                                 │
│   您没有给浙里办使用摄像头      │ ← 18sp 灰色，居中
│   的权限                        │
│                                 │
│  ┌───────────────────────────┐  │
│  │  去系统设置开启           │  │ ← 主按钮 52dp + 20sp 橙色填充
│  └───────────────────────────┘  │
│                                 │
│  ┌───────────────────────────┐  │
│  │  返回登录                 │  │ ← 次按钮 52dp + 18sp 橙色边
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

**文案**：
- 主标："无法打开摄像头"
- 说明："您没有给浙里办使用摄像头的权限"
- 主按钮："去系统设置开启"（mock：仅显示 SnackBar"请到手机系统设置 → 浙里办 → 权限 → 允许使用摄像头"，2 秒后回到登录页）
- 次按钮："返回登录"

**交互**：
- × → 返回登录页
- "去系统设置开启" → SnackBar 提示 → 2 秒后跳登录页
- "返回登录" → 跳登录页

---

### E2 · 检测超时退回登录页

**触发**：S4 / S5 / S7 任一状态 20 秒未完成。

**操作**：
1. 释放摄像头资源 + MediaPipe 实例
2. 路由跳 `context.go(AppRoutes.login)`
3. 跳转后在登录页底部显示 SnackBar，4 秒自动消失

**SnackBar 文案**：
- 主文："认证时间过长，请重新尝试"
- 颜色：橙色背景白色文字
- 字号 18sp
- 位置：登录页底部，距底 80dp（避开"登录"按钮）

**不弹任何"是否重试"对话框**——用户已经在登录页，自己选择是再点"开始认证"还是"其他方式"。

---

### E3 · 用户主动退出

**触发**：S3 / S4 / S5 / S6 / S7 / S8 任意状态点左上角 × 按钮。

**操作**：
1. 释放摄像头资源 + MediaPipe 实例（如已初始化）
2. **无二次确认弹窗**（老年人意图明确）
3. 跳 `context.pop()` 返回上一页（通常是登录页）

**视觉**：当前页淡出 200ms，登录页淡入。

---

### E4 · 模型/摄像头加载失败

**触发**：
- S3 等待 8 秒未就绪
- 或 MediaPipe 模型加载 throw 异常（CDN 拉取失败、离线）
- 或 摄像头硬件不可用

**操作**：
1. 释放已申请的资源
2. 跳 `context.go(AppRoutes.login)` 退回登录页
3. SnackBar 提示

**SnackBar 文案**：
- 主文："摄像头打不开，请稍后再试"
- 其他规范同 E2 SnackBar

## 5. 状态机表（便于 frontend 实现）

```dart
enum FaceAuthState {
  prepare,           // S0
  appOverlayShown,   // S1
  systemPermShown,   // S2
  cameraInit,        // S3
  faceAlign,         // S4
  blinkDetecting,    // S5
  blinkSuccess,      // S6 (1s pause)
  headTurnDetecting, // S7
  headTurnSuccess,   // S8 (1s pause)
  allSuccess,        // S9 (1.5s pause)
  // 异常
  permissionDenied,  // E1
  // 退出态走路由跳转，无独立 state
}
```

**状态转移触发事件**：
| 当前 | 事件 | 下一态 |
|---|---|---|
| prepare | 用户点"开始认证" | appOverlayShown |
| appOverlayShown | 用户点"同意并继续" | systemPermShown |
| systemPermShown | 用户点"允许" | cameraInit |
| systemPermShown | 用户点"禁止" | permissionDenied (E1) |
| cameraInit | 摄像头+模型就绪 | faceAlign |
| cameraInit | 8 秒超时 / 加载失败 | → E4（直接路由跳转） |
| faceAlign | 对准稳定 1 秒 | blinkDetecting |
| faceAlign | 20 秒超时 | → E2 |
| blinkDetecting | 检测到眨眼 | blinkSuccess |
| blinkDetecting | 20 秒超时 | → E2 |
| blinkSuccess | 1 秒计时结束 | headTurnDetecting |
| headTurnDetecting | 检测到摇头 | headTurnSuccess |
| headTurnDetecting | 20 秒超时 | → E2 |
| headTurnSuccess | 1 秒计时结束 | allSuccess |
| allSuccess | 1.5 秒计时结束 | → S10 路由跳转 |
| 任意（除 prepare） | × 按钮 | → E3 路由 pop |

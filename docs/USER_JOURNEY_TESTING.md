# 用户旅程图（测试版）

> **版本**：v1.0（2026-05-09）
> **作者**：pm
> **用途**：真机测试用例参考，精确到每一步用户操作和系统响应
> **覆盖场景**：7 个（含 4 个核心场景 + 超出范围 + 草稿箱恢复 + 网络断线）

---

## 如何阅读本文档

每一步格式统一：

- **用户操作**：用户具体做什么（点哪里 / 说什么 / 看什么）
- **系统响应**：WebSocket 消息类型 + 后端逻辑
- **前端表现**：用户能看到的 UI 变化、动画、文字
- **预期结果**：该步完成后的系统状态

**分支路径**用缩进标注；**异常路径**用 `⚠️` 标注；每个场景开头标注能力等级。

## 公共前置：进入长辈版首页

所有核心场景的起点。

**步骤 P1：打开 APP**
- 用户操作：点击 APP 图标
- 系统响应：渲染闪屏页，1.5 秒后自动跳转 `/home`
- 前端表现：闪屏动画 → 标准版首页（蓝色系，含"长辈版"快捷按钮）
- 预期结果：停留在标准版首页

**步骤 P2：进入长辈版**
- 用户操作：点击 Hero 区"长辈版"快捷按钮
- 系统响应：GoRouter 跳转 `/elder`
- 前端表现：页面切换为橙色大字长辈版首页
- 预期结果：长辈版首页加载完成，底部 Tab 中央可见橙色圆形助手按钮

## 场景 1a：登录 · 刷脸分支

**能力等级：L1（纯语音引导 + 视觉高亮，代理不填任何字段）**

**前置条件**：用户在长辈版首页，尚未登录

---

### 主路径

**步骤 1：唤醒代理**
- 用户操作：点击底部 Tab 中央橙色圆形助手按钮
- 系统响应：
  - 前端新建 session_id，调用 `WsClient.connect()`
  - 连接成功后发送 `agent_wake { session_id, trigger: "button", current_page: "/elder" }`
  - 后端检查草稿（当前页不在 draft 页列表，跳过）
- 前端表现：
  - 代理面板从屏幕底部滑入（300ms easeOutCubic）
  - 上方页面变暗（`rgba(0,0,0,0.4)` 遮罩）
  - 面板状态 → `listening`，状态栏文字"按住说话"
- 预期结果：面板打开，等待用户输入

**步骤 2：说出需求**
- 用户操作：按住面板底部麦克风大按钮，说"帮我登录"，松开
- 系统响应：前端逐帧发送 `audio_chunk` × N，松开后发送 `audio_end`
- 前端表现：
  - 按住期间：状态栏文字"正在听您说话…"，麦克风按钮亮橙，外圈声波扩散动画
  - 松开后：切换为"小浙正在想…"，气泡区出现"小浙正在想…"灰色占位
- 预期结果：音频发送完毕，等待 ASR 识别

**步骤 3：ASR 返回识别结果**
- 用户操作：无
- 系统响应：后端发送 `asr_result { text: "帮我登录", is_final: true }`（此消息直接在 ws_handler 内部消费，不发到前端）；后端调用 LLM 分类意图 → `scene_id: "login_face"`；生成复述话术
- 前端表现：气泡区显示"小浙正在想…"持续（对应 `agent_thinking` 状态）
- 预期结果：后端意图分类完成，开始生成复述文本

**步骤 4：代理复述确认**
- 用户操作：无（被动接收）
- 系统响应：后端发送 `agent_reply { text: "帮您登录，对吗？", requires_confirmation: true, tts_audio_base64: "..." }`
- 前端表现：
  - 气泡区"小浙正在想…"占位消失
  - 出现小浙橙色气泡"帮您登录，对吗？"
  - 气泡下方出现两个确认按钮：**橙色"对的"** / **白色"不是"**
  - TTS 播报"帮您登录，对吗？"
  - 面板状态 → `confirming`
- 预期结果：等待用户确认

**步骤 5：用户确认**
- 用户操作：点击**"对的"**按钮（或语音说"对"）
- 系统响应：前端发送 `user_confirm { answer: "yes", raw_text: "对的" }`；后端进入 executing 状态，开始按场景 prompt 调用工具
- 前端表现：
  - 气泡区新增用户气泡"对的"
  - 确认按钮消失
  - 接到 `agent_executing` → 面板收缩为迷你栏（80dp），显示"小浙正在帮您操作…" + 旋转橙色图标
- 预期结果：代理开始执行 login_face 场景

**步骤 6：代理跳转到登录页**
- 用户操作：无
- 系统响应：后端工具调用 `cmd_navigate` → 发送 `cmd_navigate { target_route: "/login", voice_hint: "好的，帮您打开登录页面" }`
- 前端表现：
  - `AgentCommandExecutor` 接到消息，GoRouter 跳转 `/login`
  - TTS 播报"好的，帮您打开登录页面"
  - 上方业务区域切换为登录页，代理面板迷你栏保持
- 预期结果：页面已在 `/login`

**步骤 7：代理高亮"同意条款"勾选框**
- 用户操作：无（被动等待）
- 系统响应：后端发送 `cmd_highlight { element_key: "chk_agree_terms", voice_hint: "请先勾选同意条款", duration_ms: 8000 }`
- 前端表现：
  - "同意条款"勾选框出现橙色脉冲高亮描边（3dp，1000ms 循环）
  - 蒙层在该区域挖空
  - TTS 播报"请先勾选同意条款"
- 预期结果：用户看到高亮提示，可点击

**步骤 8：用户点击同意条款**
- 用户操作：点击高亮的"同意条款"勾选框
- 系统响应：前端 UI 状态变更（勾选 = true），高亮停止
- 前端表现：勾选框打勾，橙色高亮消失
- 预期结果：同意条款已勾选

**步骤 9：代理高亮"刷脸登录"按钮**
- 用户操作：无
- 系统响应：后端发送 `cmd_highlight { element_key: "btn_face_login", voice_hint: "请点这个按钮开始刷脸", duration_ms: 8000 }`
- 前端表现：
  - "刷脸登录"按钮出现橙色脉冲高亮
  - TTS 播报"请点这个按钮开始刷脸"
- 预期结果：等待用户点击

**步骤 10：用户点击刷脸按钮**
- 用户操作：点击高亮的"刷脸登录"按钮
- 系统响应：页面自身逻辑处理（浙里办内部），代理不介入
- 前端表现：高亮消失，页面跳转到刷脸验证流程
- 预期结果：进入刷脸验证页面 `/login/face`

**步骤 11：代理导航到刷脸页并语音引导**
- 用户操作：无
- 系统响应：后端发送 `cmd_navigate { target_route: "/login/face", voice_hint: "请正对屏幕，保持不动，开始刷脸" }`
- 前端表现：
  - TTS 播报"请正对屏幕，保持不动，开始刷脸"
  - 上方显示刷脸验证页面（摄像头画面）
- 预期结果：刷脸页就绪，用户自行完成刷脸

**步骤 12：用户完成刷脸，任务结束**
- 用户操作：对着摄像头完成刷脸（眨眼 / 摇头等）
- 系统响应：浙里办系统验证成功；后端发送 `task_done { summary: "已引导您完成刷脸登录...", voice_hint: "...", tts_audio_base64: "..." }`
- 前端表现：
  - 面板从迷你栏展开，显示任务完成摘要
  - TTS 播报完成话术
  - TTS 播完 + 1 秒后，面板向下滑出关闭
  - 操作记录写入 IndexedDB
- 预期结果：面板关闭，用户已登录

### 分支路径 A：用户拒绝复述确认（步骤 5 说"不是"）

**步骤 5-B：用户点"不是"**
- 用户操作：点击**"不是"**按钮
- 系统响应：前端发送 `user_confirm { answer: "no" }`；后端不执行任何场景工具，重新进入 listening 状态
- 前端表现：
  - 气泡区新增用户气泡"不是"
  - 确认按钮消失
  - 状态切回 listening，可重新说话
  - 小浙气泡出现"好的，您想做什么？"（由后端发送 agent_reply）
- 预期结果：代理不执行任何操作，等待用户重新说需求

---

### 异常路径 ⚠️：ASR 没听清

**在步骤 2 后触发**：

**步骤 2-E：ASR 识别失败**
- 系统响应：后端发送 `agent_error { error_code: "asr_unclear", retry_count: 1, voice_hint: "没听清，请再说一次", tts_audio_base64: "..." }`
- 前端表现：
  - 气泡区出现小浙气泡"没听清，请再说一次"
  - TTS 播报同内容
  - 面板状态 → `listening`，麦克风按钮重新可用
- 预期结果：用户可再次按住麦克风重试（最多 3 次；第 3 次失败后代理说"您也可以试试文字输入"，状态 → `idle`）

## 场景 1b：登录 · 验证码分支

**能力等级：L2（代读短信验证码 + 代填，"登录"按钮用户亲按）**

**前置条件**：用户在刷脸验证页 `/login/face`，已唤醒代理或从刷脸失败后触发

---

### 主路径

**步骤 1：唤醒代理（同场景 1a 步骤 1-4）**

略，同前。用户说"帮我用验证码登录"，复述确认"帮您用验证码登录，对吗？"，用户点"对的"。

**步骤 2：代理跳转到验证码页**
- 系统响应：后端发送 `cmd_navigate { target_route: "/login/verify", voice_hint: "好的，帮您打开验证码登录页面" }`
- 前端表现：GoRouter 跳转 `/login/verify`，TTS 播报
- 预期结果：验证码登录页就绪

**步骤 3：代理高亮手机号输入框**
- 系统响应：`cmd_highlight { element_key: "input_phone", voice_hint: "请输入您的手机号", duration_ms: 8000 }`
- 前端表现：手机号输入框橙色高亮，TTS 播报"请输入您的手机号"
- 预期结果：等待用户输入手机号

**步骤 4：用户输入手机号**
- 用户操作：点击高亮的输入框，用键盘输入手机号
- 系统响应：前端本地 UI 变化，无 WS 消息
- 预期结果：手机号已填入

**步骤 5：代理高亮"发送验证码"按钮**
- 系统响应：`cmd_highlight { element_key: "btn_send_code", voice_hint: "请点击发送验证码", duration_ms: 8000 }`
- 前端表现：发送验证码按钮橙色高亮，TTS 播报
- 预期结果：用户看到提示

**步骤 6：用户点击"发送验证码"**
- 用户操作：点击高亮的"发送验证码"按钮
- 系统响应：页面逻辑向手机发送短信；代理继续下一步
- 预期结果：短信已发送

**步骤 7：代理请求 read_sms 授权（HITL 暂停）**
- 系统响应：
  - 后端工具 `read_sms` 设有 `stop_after_tool_call=True`，Agno Agent 暂停
  - 后端发送 `permission_request { permission_id: "xxx", permission_type: "read_sms", description: "小浙需要读取您的短信验证码，是否授权？", expires_in_ms: 20000 }`
- 前端表现：
  - 面板从迷你栏展开
  - 气泡区出现授权卡片：标题"需要授权"，正文"小浙需要读取您的短信验证码，是否授权？"
  - 两个按钮：**橙色"可以"** / **白色"不行"**
  - TTS 播报授权请求话术
- 预期结果：等待用户当场授权

**步骤 8：用户同意授权**
- 用户操作：点击**"可以"**按钮
- 系统响应：前端发送 `permission_response { granted: true, input_mode: "touch", raw_text: "可以" }`；后端 HITL 解除，继续执行 `read_sms` 工具
- 前端表现：授权卡片消失，面板迷你栏"小浙正在帮您操作…"
- 预期结果：代理获得授权，继续读取验证码

**步骤 9：代理代填验证码**
- 系统响应：后端发送 `cmd_fill_field { field_key: "input_verify_code", field_label: "验证码", value: "123456", is_sensitive: false, voice_hint: "已帮您填入验证码" }`
- 前端表现：
  - 验证码输入框逐位出现数字（填充动画）
  - TTS 播报"已帮您填入验证码"
- 预期结果：验证码字段已填写完毕

**步骤 10：代理高亮"登录"按钮（止损点）**
- 系统响应：`cmd_highlight { element_key: "btn_login", voice_hint: "请点击登录", duration_ms: 8000 }`
- 前端表现：
  - "登录"按钮橙色高亮
  - TTS 播报"请点击登录"
  - **代理到此止步，不代按**
- 预期结果：用户亲手完成最后一步

**步骤 11：用户亲手点"登录"**
- 用户操作：点击高亮的"登录"按钮
- 系统响应：页面验证登录，成功后后端发送 `task_done`
- 前端表现：TTS 播报完成话术，面板 TTS 播完 + 1 秒后关闭
- 预期结果：登录成功，面板关闭

### 分支路径 B：用户拒绝 read_sms 授权（步骤 8）

**步骤 8-B：用户点"不行"**
- 用户操作：点击**"不行"**按钮
- 系统响应：前端发送 `permission_response { granted: false }`；后端中止 read_sms，发送 `agent_reply { text: "好的，已取消", ... }`
- 前端表现：
  - 授权卡片消失
  - 小浙气泡"好的，已取消"
  - 面板状态 → `listening`
- 预期结果：代理不读短信，流程中止；用户可自行输入验证码或重新发起任务

---

### 异常路径 ⚠️：授权超时（20 秒内无响应）

- 系统响应：`agent_core.py` 的 `wait_for_permission` 超时 → 返回 `False` → 后端发送 `agent_reply { text: "好的，已取消" }`
- 前端表现：同"拒绝授权"路径
- 预期结果：代理不执行 read_sms

## 场景 6：网络断线重连

**前置条件**：代理面板已打开，用户正在使用中

---

### 主路径

**步骤 1：网络断开**
- 触发条件：设备 Wi-Fi 断开 / 服务器重启 / 网络异常
- 系统响应：`WsClient` WebSocket 连接断开；`_session.websocketConnected = false`
- 前端表现：
  - 面板顶部出现红色横幅："网络已断开，小浙暂时无法帮您"（18sp 白色字）
  - 麦克风按钮变灰，不可点击
  - 文字输入框禁用
- 预期结果：用户知晓网络断开，面板 UI 锁定

**步骤 2：用户等待或主动恢复网络**
- 用户操作：打开 Wi-Fi / 切换网络
- 系统响应：当前实现**无自动重连**——`WsClient` 断开后不自动重试
- 前端表现：网络横幅持续显示，不自动消失
- 预期结果：用户需手动关闭面板再重新打开

**步骤 3：用户关闭面板重新打开**
- 用户操作：点击面板右上角 × 关闭，再次点击助手按钮
- 系统响应：`AgentPanel` 重新初始化，调用 `WsClient.connect()`
- 前端表现：面板重新从底部滑入，建立新连接，红色横幅不再出现
- 预期结果：新 session 建立，可正常使用

---

### 网络中途断开的状态残留

**注意**：若代理正在执行任务（executing 状态）时网络断开：
- 前端：面板顶部红色横幅叠加在迷你栏上
- 后端：session 丢失，半途的工具调用中止
- 已填字段：若 `DraftService.autoSave` 正常工作，已填字段已写入 IndexedDB，用户下次打开有草稿恢复提示（仅限有草稿功能的页面）
- 未保存的字段：丢失

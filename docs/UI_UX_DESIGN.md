## 1 交互设计总览

### 1.1 现有页面结构梳理

基于浙里办截图与场景画布 v1，长辈版页面体系保留如下：

| 页面 | 处理方式 | 说明 |
|---|---|---|
| 标准版首页 | 保留，仅做长辈版入口 | 右上角"长辈版"按钮，其余不动 |
| 长辈版首页（热门服务/我的常用/我的订阅） | 保留结构，新增"助手"按钮 | 底部 Tab 栏中间原"搜索"麦克风位替换为"助手"按钮 |
| 登录页面 | 保留，不做功能改动 | 代理在此页以语音引导为主（L1） |
| 刷脸身份验证页面 | 保留 | 代理只做语音解释，不介入系统弹窗 |
| 验证码页面 | 保留 | 代理可代读短信并代填验证码（L2） |
| 医保缴费页面 | 保留"我为自己缴"表单结构 | 代理逐字段引导填写（L2-L3） |
| 医保查询 / 养老金查询页面 | 保留社保查询结构 | 代理可全链路完成（L1-L2） |
| 搜索页面 | 保留，底部 Tab 可达 | 代理不主动介入搜索流程 |
| "我的"页面 | 保留原有功能入口，**新增"操作记录"和"草稿箱"入口** | 替换"我的草稿"位置，改为代理专属草稿箱 |

**新增页面**：
- 操作记录详情页（子女查看用）

**移除**：无页面移除，保持浙里办原有导航骨架。

### 1.2 代理"小浙"在页面体系中的位置

**"助手"按钮位置**：底部 Tab 栏中央，替代原浙里办的麦克风/搜索按钮位置。

```
┌──────────────────────────────────────┐
│           长辈版页面内容区              │
│                                      │
│                                      │
│                                      │
├──────────────────────────────────────┤
│  ★首页    【浙】助手    👤 我的        │  ← 底部 Tab 栏
└──────────────────────────────────────┘
```

- "助手"按钮：橙色圆形，中央"浙"字卡通小人图标，直径 56dp
- 点击"助手"按钮 → 代理面板从底部滑上
- **代理面板叠加在以下页面上**：长辈版首页、医保缴费页、医保查询页、养老金查询页、登录页、刷脸验证页、验证码页
- 登录流程页面（刷脸、验证码）也保留底部 Tab 栏中的"助手"按钮入口

### 1.3 草稿箱与操作记录入口

**草稿箱**：
- 入口位置："我的"页面，原"我的草稿"位置，改标签为"草稿箱"
- 状态徽章：有未完成草稿时，图标右上角显示红色数字徽章（如"2"），由前端启动时读取 IndexedDB 获得 N 值
- 图标：编辑文档图标（橙色），字号 20sp

**操作记录**：
- 入口位置："我的"页面，原"办事记录"位置，改标签为"操作记录"
- 子标题文字："小浙帮您做了什么，一目了然"（18sp 灰色）

## 2 代理面板 UI 设计

### 2.1 面板尺寸与结构

面板占屏幕高度 **45%**（约 380dp on 844dp 高度屏幕），从底部滑上，上方页面仍可见。

```
┌──────────────────────────────────────┐
│                                      │
│         上方业务页面（变暗 40%）        │  ← 55% 区域，可见不可交互
│                                      │
├──────────────────────────────────────┤ ← 拖拽条（灰色 4dp 圆角横条）
│  [浙] 小浙    ×                       │  ← 面板标题栏（48dp 高）
├──────────────────────────────────────┤
│                                      │
│  对话气泡区（可滚动）                   │  ← 约 180dp
│  ┌─────────────────────────────┐     │
│  │ 小浙：有什么可以帮您？         │     │
│  └─────────────────────────────┘     │
│                                      │
├──────────────────────────────────────┤
│  [状态行] 正在听您说话… 🎙️            │  ← 32dp 高，动态文字
├──────────────────────────────────────┤
│        ●  按住说话                    │  ← 麦克风大按钮（72dp 圆形）
│   （松开发送，滑上取消）               │
└──────────────────────────────────────┘
```

### 2.2 面板内部布局细节

| 区域 | 规格 |
|---|---|
| 拖拽条 | 宽 40dp，高 4dp，圆角 2dp，颜色 `#DDDDDD`，居中 |
| 标题栏 | 高 48dp；左侧小浙卡通图标 32dp + "小浙"文字 20sp 粗体；右侧 × 关闭按钮 44×44dp 触控区 |
| 对话气泡区 | 高约 180dp，可竖向滚动；小浙气泡左对齐（橙色 `#FFF0E6` 底，`#1A1A1A` 文字，圆角 16dp，最大宽度 80%）；用户语音识别文字右对齐（白底灰描边） |
| 状态行 | 高 32dp，居中，字号 16sp，颜色 `#888888` |
| 麦克风按钮 | 直径 72dp 圆形，主色 `#FF6D00`，白色麦克风图标 32dp；触控区 88dp（含外圈）；按住变深色 `#CC5700` |

### 2.3 面板打开与关闭动画

| 操作 | 动画 |
|---|---|
| 打开（点击"助手"按钮） | 面板从屏幕底部向上滑入，时长 300ms，easeOutCubic 缓动；同时上方页面背景渐变变暗（300ms） |
| 关闭（点击 × 按钮） | 面板向下滑出，时长 250ms，easeInCubic；背景同步还原 |
| 关闭（任务完成自动收起） | 同上，延迟 1500ms 后自动关闭（留时间让用户听到完成语音） |
| 用户下拉面板 | 拖拽超过 80dp 后松手自动关闭 |

### 2.4 面板与上方页面的关系

- 上方页面**可见，但覆盖半透明黑色蒙层（40% 透明度，`rgba(0,0,0,0.4)`）**
- 上方页面**不可交互**（蒙层拦截触摸事件），防止老年用户误触
- 代理执行高亮指令时，蒙层在对应元素区域挖空（透明度降为 0%），使高亮元素透出可点击

### 2.5 授权弹窗设计

当代理需要一事一授权时（如代填身份证号），在面板内部弹出授权卡片，**不脱离面板**：

```
┌─────────────────────────────────────┐
│  ⚠️  小浙想帮您填                    │
│                                     │
│       【 身份证号 】                  │  ← 24sp 粗体，橙色
│                                     │
│  这次可以吗？                         │  ← 20sp
│                                     │
│  ┌──────────┐    ┌──────────┐       │
│  │  不用了   │    │  可以    │       │  ← 各 56dp 高，最小 120dp 宽
│  └──────────┘    └──────────┘       │
│   灰色描边底         橙色填充底        │
└─────────────────────────────────────┘
```

- 弹出动画：从面板底部向上弹出，200ms
- "可以"按钮：`#FF6D00` 底色，白色文字，圆角 12dp
- "不用了"按钮：白底，`#888888` 描边，`#333333` 文字，圆角 12dp
- 同时 TTS 播放授权请求话术（对应 `permission_request` 消息）
- 超时 15 秒无响应 → 自动视为拒绝，代理语音说"好的，这一步我就不帮您填了"

### 2.6 适老化要素清单

- 小浙卡通图标：橙色圆形底，白色"浙"字小人，简洁不复杂
- 所有可点击元素触控区 ≥ 48dp
- 对话气泡文字 ≥ 20sp
- 面板内无超过 2 层的交互嵌套

### 3.1 场景 1：登录 · 刷脸分支（L1 纯引导）

- 用户说"要" → 代理跳转回医保缴费页并恢复草稿字段

### 3.4 场景 4：养老金查询（L1-L2 全链路，含结果双通道播报）

**起点**：用户在长辈版首页，点击底部"助手"按钮
**终点**：查询结果以语音 + 屏幕双通道呈现，面板延迟关闭

| 步骤 | 用户动作 | 屏幕变化 | 代理语音 | 消息类型 |
|---|---|---|---|---|
| 1 | 点击底部"助手"按钮 | 代理面板滑上 | "您好，我是小浙，有什么可以帮您？" | `agent_wake` → `agent_ready` |
| 2 | 说"帮我查养老金" | 状态行"正在听您说话…" | — | `audio_chunk` / `audio_end` |
| 3 | — | 思考动画；气泡"帮您查养老金，对吗？" | "帮您查养老金，对吗？" | `asr_result` → `agent_reply` |
| 4 | 说"对" | — | — | `user_confirm`（yes） |
| 5 | — | 页面跳转到养老金查询页 | "好的，帮您打开养老金查询" | `cmd_navigate`（/elder/pension-query） |
| 6 | — | 查询条件字段（若有）自动填入；"查询"按钮高亮橙色（1 秒）后自动按下 | "帮您查一下" | `cmd_fill_field`（查询条件）→ `cmd_press_button`（btn_query，is_deterministic: false） |
| 7 | — | 页面渲染查询结果（养老金数据卡片显示在上方业务页面）；面板气泡同步出现结果摘要文字 | "您的养老金本月发放了 XXX 元，已经到账了" | `agent_reply`（含 tts_audio_base64）；前端同时触发结果页渲染 |
| 8 | — | 面板 3000ms 后自动关闭（给用户时间听完播报并看结果） | — | `task_done` |

**双通道说明**：
- 屏幕通道：查询结果在上方业务页面区域展示（数据卡片，大字号）
- 语音通道：TTS 播报关键数字（金额、日期）
- 两者同步触发，互为补充

## 4 适老化设计规范

### 4.1 色彩规范

| 用途 | 色值 | 说明 |
|---|---|---|
| 长辈版主色 | `#FF6D00` | 按钮、激活 Tab、代理相关强调色 |
| 代理面板底色 | `#FFFFFF` | 纯白，保持清晰 |
| 代理气泡背景 | `#FFF0E6` | 浅橙色，区分小浙发言 |
| 用户气泡背景 | `#F5F5F5` | 浅灰，区分用户输入 |
| 高亮元素描边 | `#FF6D00` | 脉冲高亮描边色 |

## 5 特殊状态 UI

### 5.1 "正在听您说话"状态

- **触发时机**：用户按住麦克风按钮（或按钮变为激活态）
- **视觉**：
  - 状态行文字变为"正在听您说话…"，橙色 `#FF6D00`
  - 麦克风按钮外圈出现橙色声波扩散动画（3 圈，从内到外渐隐，周期 1200ms）
  - 面板顶部出现橙色细进度条（indeterminate 动画）
- **对应架构**：前端本地感知麦克风激活状态，不依赖后端消息；发送 `audio_chunk` 期间持续显示

### 5.2 "正在想…"等待状态

- **触发时机**：`asr_result` 返回后，200ms 内未收到 `agent_reply`
- **视觉**：
  - 状态行文字变为"小浙正在想…"，灰色
  - 小浙气泡区出现三点跳动动画（●●●，每点延迟 200ms，循环）
  - 麦克风按钮变灰不可点击（防止用户重复发话）
- **超时处理**：8 秒后显示错误状态（见 5.4）

### 5.3 网络断开提示 UI

- **触发时机**：WebSocket 连接断开（`websocket_connected: false`）
- **视觉**：
  - 代理面板顶部出现橙红色横幅（高 40dp）："网络已断开，小浙暂时无法帮您"
  - 横幅背景 `#FF3B30`，文字白色 18sp
  - 麦克风按钮变灰，不可点击
  - 状态行显示"等待网络恢复…"
- **恢复时**：横幅自动消失，按钮恢复，状态行回到默认"有什么可以帮您？"

### 5.4 "听不清"重试 UI

- **触发时机**：收到 `agent_error`（error_code: asr_unclear）
- **第 1-2 次重试**：
  - 小浙气泡显示"不好意思没听清，您再说一遍"
  - TTS 播报同内容
  - 麦克风按钮自动重新激活（橙色），等待用户再次按住
- **第 3 次仍失败**：
  - 小浙气泡显示"您也可以试试手动操作，我先退下了"
  - TTS 播报
  - 面板自动关闭（延迟 2000ms）

### 5.5 "这个我不会"超出能力 UI

- **触发时机**：收到 `agent_out_of_scope`
- **两种子情况**：

| 情况 | 小浙气泡文字 | TTS | 后续 |
|---|---|---|---|
| 浙里办没有该服务（如"订火车票"） | "浙里办没有订火车票的服务，您可以试试 12306" | 同文字 | 面板保持，等用户再提需求 |
| 浙里办有但代理不会（如"改头像"） | "改头像这个功能在"我的"页面里，我暂时帮不了您，您可以自己试试" | 同文字 | 同时高亮对应入口（若能定位到） |

- 视觉：气泡左上角显示小浙图标 + 灰色"我帮不上"标签（12sp），区分普通回复

## 附录 A：消息类型与 UI 状态对应速查

| WebSocket 消息类型 | 触发的 UI 变化 |
|---|---|
| `agent_wake` | 前端发送，无 UI 变化（等 agent_ready） |
| `agent_ready` | 面板打开，播报问候语，若有 draft_hint 显示草稿提示气泡 |
| `audio_chunk` / `audio_end` | 麦克风激活动画，状态行"正在听您说话…" |
| `asr_result` | 用户气泡出现识别文字；状态行切换为"小浙正在想…" |
| `agent_reply` | 小浙气泡出现，TTS 播放；若 requires_confirmation 则显示"是 / 不是"两按钮 |
| `user_confirm` | 前端发送，隐藏确认按钮 |
| `cmd_navigate` | 页面跳转，voice_hint 触发 TTS |
| `cmd_highlight` | 目标元素描边脉冲，蒙层挖空，TTS 播放 voice_hint |
| `cmd_fill_field` | 目标字段填充动画，TTS 播放 voice_hint |
| `cmd_press_button` | 目标按钮触发点击（前端白名单校验），TTS 播放 voice_hint |
| `permission_request` | 面板内弹出授权卡片，TTS 播放授权话术 |
| `permission_response` | 前端发送，授权卡片收起 |
| `task_done` | TTS 播放完成话术，1500ms 后面板自动关闭，写操作记录 |
| `agent_error`（asr_unclear） | 小浙气泡显示"没听清"，麦克风重新激活 |
| `agent_out_of_scope` | 小浙气泡显示超出能力回复，TTS 播放 |

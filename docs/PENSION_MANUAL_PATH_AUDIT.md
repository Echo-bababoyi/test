# 养老金查询 — 纯手动流程审查

> **日期**：2026-05-19 ｜ **作者**：architect ｜ **范围**：不考虑智能代理，只看用户手动路径

---

## 1. 结论速览

**手动流程"几乎"能跑通，但有 3 个具体问题需要修**：

1. 🟠 **长辈版首页无直接入口** —— 用户从首页只能通过"我的常用 Tab → 社保查询 → 养老保险卡 → 基本信息" **三跳**才能到达养老金查询页，违背用户旅程"直接说一句话就能看到结果"的预期，且对手动操作用户极不友好
2. 🟠 **月份选择器形同摆设** —— 左右箭头 `onTap: () {}` 是空占位，用户**无法切换月份**，只能默认 "2026年5月"
3. 🟡 **多处字号低于适老化 18sp 阈值** —— 引导文字、个人信息卡、详情行有 5+ 处 14-17sp

`_doQuery` 本身是纯本地 mock（不依赖后端），点击后能正常渲染结果。**只要进得来这页 + 点查询，结果展示是 OK 的**。

### 2.3 `_doQuery`（行 28-38）

```dart
void _doQuery() {
  setState(() => _hasResult = true);
  WsClient.instance.send('query_result_ready', {
    'page_id': 'pension_query',
    'result_fields': {
      'month': _selectedMonth,
      'amount': _mockAmount,   // '3280'
      'unit': '元',
    },
  });
}
```

- **纯本地 mock**：`setState(_hasResult=true)` 触发个人信息卡 + 金额卡渲染，金额硬编码 `_mockAmount = '3280'`
- 不调任何后端 API
- `WsClient.send` 只是通知后端"结果已渲染"，让代理可以播报。**WS 未连接时静默 skip**（ws_client.dart:52-55 `send skipped (not connected)`），不影响本地渲染
- ✅ **纯手动流程不依赖后端**，无 WS 也能跑

### 2.4 结果展示

- 个人信息卡：橙渐变 + "已认证"角标 + 姓名"*宇澄"+ 证件号脱敏；mock，**展示合理**

## 3. 问题清单与修复方案

### 🟠 P0：长辈首页加直接入口

**问题**：当前用户找养老金查询要 3 跳，违反 PRD 场景 4 用户故事"不用在各种菜单里翻找入口"。

**修复方向**（与 PM 拍板二选一）：

**选项 A（推荐）**：把"线上一站办"格栅中某项绑到养老金查询路由。当前 `_items`（elder_home.dart:575-582）：
```
健康医保 / 社会保障 / 行驶驾驶 / 身份户籍 / 文旅体育 / 查看全部
```
其中**"社会保障"**最贴近——把它的 onTap 从 `_showTodo` 改成 `context.push(AppRoutes.pensionQuery)`，或拆出独立"养老金查询"格子（取代"文旅体育"）。

**选项 B**：把"我的常用" Tab（elder_home.dart:423-459）现在的 2 张卡（浙里医保 / 社保查询）扩展为 3 张，加一张"养老金查询"，直接 `context.push(AppRoutes.pensionQuery)`。

**选项 C**：把"授权办"格栅里那个"养老保险年限"（elder_home.dart:869）的 onTap 从 `_showTodo` 改成 `context.push(AppRoutes.pensionQuery)`。**最小改动 1 行**，但语义稍弱（"年限"≠"金额"）。

**建议选 B**：与社保查询并列，一致性最好；改动量也小（在 _EldFavoritesContent 加一个 `_EldServiceCard`）。

### 🟠 P0：月份选择器箭头加功能

**`pension_query_page.dart:140-153`** 两个箭头 `onTap: () {}` 是空。

**最小修复**（仅切月份字符串，不影响 mock 数据）：

```dart
// 加常量列表（约第 23 行 _mockAmount 之后）
static const _months = [
  '2026年5月', '2026年4月', '2026年3月',
  '2026年2月', '2026年1月', '2025年12月',
];

void _prevMonth() {
  final i = _months.indexOf(_selectedMonth);
  if (i < _months.length - 1) setState(() => _selectedMonth = _months[i + 1]);
}
void _nextMonth() {
  final i = _months.indexOf(_selectedMonth);
  if (i > 0) setState(() => _selectedMonth = _months[i - 1]);
}
```

行 141：`onTap: () {}` → `onTap: _prevMonth`
行 151：`onTap: () {}` → `onTap: _nextMonth`

**附带提升触控区**（适老化）：把 `Icon(size: 24)` 包成 `SizedBox(width: 44, height: 44, child: Center(child: Icon...))`，触控目标达到适老化 44dp。

### 🟢 P2：可选改进

- `_selectedMonth = '2026年5月'`（行 22）写死；如果改月份选择器有效，可初始化为当前月 `'${DateTime.now().year}年${DateTime.now().month}月'`
- 不同月份显示不同 mock 金额（如 5 月 3280、4 月 3250、3 月 3200）让换月有"真实感"——但论文里需要解释清楚是 mock
- 标准版 / 长辈版字号 token 化：design_tokens.dart 里已有 `AppFontSize.elderBody=18 / elderTitle=24 / elderLarge=32`，**这页应使用这些 token**而不是硬编码数字——既保证一致性，也方便后续统一调整

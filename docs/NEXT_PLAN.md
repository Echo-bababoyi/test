## 用户需要做的事（开发团队做不了的）

### U1 · 讯飞账号申请（对应 N1）

**操作步骤**：
1. 访问 [open.xfyun.cn](https://open.xfyun.cn) 注册账号
2. 完成实名认证（身份证信息）
3. 创建应用，开通「实时语音转写」服务
4. 获取 `APPID`、`APIKey`、`APISecret`，发给开发团队写入 `.env`

**为什么要提前做**：审核需 1-2 个工作日，是整个语音链路的关键路径。越早申请越好。

### U2 · 云服务器购买（对应 N6 前半段）

**操作步骤**：
1. 选择云厂商（推荐阿里云/腾讯云学生优惠）
   - 配置：2C2G，Ubuntu 22.04 LTS，带宽 3Mbps 以上
   - 节点：上海或杭州（延迟最低）
   - 预算：约 ¥30-60/月，毕设期间 1-2 个月即可
2. 购买后获取服务器公网 IP 和 root 密码，发给开发团队

**注意**：如果不购买域名，仍可部署，但 iOS Safari 麦克风权限受限（演示用 Android 手机可绕过）。

### N3 · 前端真实麦克风录音

| 属性 | 值 |
|---|---|
| **归属** | 开发 |
| **依赖** | 无（可立即开始） |
| **预估工时** | 3-4 小时 |
| **可并行** | 与 N2 并行 |

**内容**：

在 `app/lib/services/audio_recorder.dart` 中实现真实录音（当前后端已有 ASR 适配器接口，前端尚未接入真实音频）：

- Flutter Web：通过 `dart:js_interop` 调用 `getUserMedia` + `AudioWorklet`（iOS Safari 16.4 以下降级到 `ScriptProcessorNode`）
- Flutter Android APK：使用 `record` 插件（pub.dev，支持 Android/iOS/Web 多平台）

**音频参数**：采样率 16000 Hz，单声道，PCM 16-bit，每帧 40ms（1280 字节），base64 编码后发送 `audio_chunk`。

**iOS Safari 关键处理**：`AudioContext` 初始化必须在用户手势回调（mic 按钮 `onTap`）中触发，不能在页面加载时自动初始化。

**验收**：按住麦克风说话，后端日志出现 `audio_chunk` 消息，ASR 返回识别文字并显示在面板。

### N4 · Web Speech API 兜底正式化

| 属性 | 值 |
|---|---|
| **归属** | 开发 |
| **依赖** | N3 完成后 |
| **预估工时** | 1-2 小时 |
| **目的** | N1 讯飞审核未通过时，有完整可演示的备用方案 |

**内容**：

当环境变量 `ASR_BACKEND=web_speech` 时，后端跳过讯飞 ASR，改为前端使用 `SpeechRecognition` API 直接识别，将识别文字通过 `text_input` 消息类型发给后端（该消息类型已实现）。

实现方式：前端检测 `SpeechRecognition` API 可用性，若可用且配置为 web_speech 模式，则跳过 `audio_chunk` 流程，直接走 `text_input` 通道；UI 上麦克风按钮行为不变，用户无感知。

**注意**：Web Speech API 在 APK 中不可用（仅 Chrome 浏览器支持），APK 路径必须走 N3 真实录音 + N5 讯飞 ASR。

**最快上手路径（0 成本，无需等审核）**：

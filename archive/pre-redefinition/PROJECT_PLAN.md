# 项目开发计划

## 一、项目定位

- **当前阶段目标**：用 Flutter 完成一个**可运行的 demo**，像素级还原浙里办长辈版 32 个页面对应的所有页面、弹窗与关键动效。
- **长期定位**：作为后续**多模态交互 + 智能代理**研究的底座。复刻本身不是终点，是对照基线。
- **毕设交付物**：一份能被指导教师在真实设备（或模拟器）上点击演示的 Flutter 工程，以及与原版的还原对照材料。

## 二、范围与非目标

### 必须包含
- `docs/复刻浙里办/截图/` 中 32 张截图覆盖的所有页面与状态
- 两套主题：标准版（`#2D74DC`）与长辈版（`#FF6D00`）
- 登录两条分支：刷脸主分支、验证码备选分支
- 两类弹窗行为区别：系统弹窗（阻塞）vs 应用内弹窗（非阻塞）
- 刷脸验证动画：眨眼/摇头 + 屏幕变色
- 长页滚动拼接（长辈版首页 & 我的页面各 3 段）

### 明确不做
- 不接入真实后端——所有数据走假数据/mock
- 不做复刻阶段的"改进"（图标、间距、流程看起来过时也保持原样）
- 不做性能/包体积优化，除非影响目标设备流畅度
- 不在复刻阶段引入个人创新（多模态/智能代理留到底座之上单独阶段）
- **不出原生 Android/iOS 安装包**，不装模拟器，不做平台适配——仅 Chrome 浏览器可跑即可

## 三、架构原则（为底座留扩展接口）

以下 6 个扩展点是论文创新方向一定会动的地方。复刻时**仅建接口 + mock 实现**，不做真实现，工作量几乎不增，省掉后续一次架构重写。

| 扩展点 | 复刻时 | 后续接入 |
|---|---|---|
| `VoiceInputService` | 搜索页麦克风按钮走抽象接口，mock 返回固定文字 | 真 STT / 唤醒词 / 方言识别 |
| `FaceAuthService` | 抽象接口 + mock 动画实现 | ML Kit 活体检测 / 代理跳过 |
| `AppIntent` 意图层 | 所有页面跳转通过 intent 命令触发，`go_router` 只是执行器 | 智能代理发同样的 intent 代用户操作 |
| 交互日志 | 从第一天记录点击/停留/回退（写本地文件） | 论文可用性数据、代理决策输入 |
| `Semantics` 无障碍语义 | 每个交互组件挂语义标签 | TTS、屏幕阅读器、代理感知 UI |
| `ServiceRepository` | 社保/养老金数据走 Repository 模式 + 假数据 | 接真接口或代理生成内容 |

## 五、核心概念（实现时不可省略的区分）

1. **两种模式不是主题开关** — 标准版 `#2D74DC` 蓝，长辈版 `#FF6D00` 橙；通过专用入口按钮切换，不是设置项切换。
2. **两类弹窗在代码里必须是两套组件族** — `SystemDialog`（全屏遮罩 + 阻塞）与 `InAppOverlay`（底部浮层 + 可手势关闭）**物理分开**，不共用基类传参，避免非阻塞弹窗被误实现为阻塞。
3. **两条登录分支最终都落在长辈版主页** — 刷脸主分支：同意条款 → 刷脸请求弹窗（应用内）→ 摄像头权限（系统）→ 眨眼/摇头动画；验证码分支：其他认证方式 → 验证码页面 → 验证码系统弹窗。
4. **长页滚动是单页面多段拼接** — 长辈版首页与"我的"页的 3 段截图是同一页面的上下滚动，不是独立路由。

## 六、分阶段计划

### Phase 0：项目骨架（1–2 天）
- Flutter 项目初始化，按 feature 切分目录
- 状态管理、路由、主题系统、设计 token 落地
- 两套弹窗组件族骨架
- 11 个页面占位 Scaffold，路由全连通
- **验收**：能启动，能从每个入口跳到每个页面（内容是空壳）

### Phase 1：主线黑白线框版（3–5 天）
- 端到端跑通：启动 → 标准首页 → 长辈版首页 → 登录（刷脸）→ 搜索「医保缴费」→ 社保服务页
- 页面用灰块 + 中文标签占位，**不贴皮**
- 弹窗按两类正确分流，状态联通（登录态 / 当前模式 / Tab / 搜索框）
- **验收**：主线能点通，逻辑正确但丑；作为答辩兜底版本

### Phase 2：逐页像素级还原（约 2 周）
- 按截图一页一页贴皮
- 每页流程：拾色 → 量间距 → 抠图标 → 对照调整 → 同尺寸对比图存档
- 推荐顺序（影响面大到小）：标准首页 → 长辈首页热门 Tab → 长辈首页另两 Tab → 长辈首页 3 段滚动 → 我的页面 + 3 段滚动 → 登录 + 同意条款 → 刷脸三页 → 搜索三状态 → 搜索结果两页 → 社保三页 → 养老金查询 → 验证码分支
- **验收**：每页都有一张"原图 vs 复刻"的对照截图

### Phase 3：动效与备选分支（3–5 天）
- 刷脸验证动画：眨眼/摇头 + 屏幕变色
- 验证码短信登录分支完整流程
- 长辈首页三 Tab 切换动效、底部导航切换
- **验收**：全部动态过程与原版相符

### Phase 4：验收与冻结（2–3 天）
- 32 张对照截图全部生成并通过检查
- 过完整的还原度检查清单（见 §七）
- 低端机/模拟低端配置跑一遍
- 打 `v1.0-baseline` tag，这就是后续创新的底座
- **验收**：可交付的 demo + 对照材料包
